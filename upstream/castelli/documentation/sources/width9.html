WIDTH:


WIDTH is an abundance analysis program. It was written by R.L.Kurucz.
The original version is available at http://kurucz.harvard.edu/programs/width/

A slightly modified version which runs under LINUX  is:

1) width9.for--to be compiled with the INTEL fortran 
ifort -double-size 64 -save -o width9.exe width9.for 

2) width9g.for--to be compiled with the linux gfortran 
gfortran -fno-automatic -w -O3 -o width9g.exe width9g.for


The results from width9.for and width9g.for are the same, only the fortran is a little bit different in two codes.

INPWIDTH:

An interactive code to generate the input for WIDTH without to be
worried about its format is inpwidth.for. It was written by Fiorella Castelli.


Here are the Instructions to better understand WIDTH and INPWIDTH.
The paper with the instructions was published on Memorie SAIt, Supplements.


To compile INPWIDTH with the INTEL fortran:


ifort -save -o inpwidth.exe inpwidth.for

To execute INPWIDTH type:  

source inpwidth.com

an example of inpwidth.com is:

ln -s ~/lines/gf0100.100  fort.10
ln -s ~/lines/gf0150.100  fort.11
ln -s ~/lines/gf0200.100  fort.12
ln -s ~/lines/gf0300.100  fort.13
ln -s ~/lines/gf0400.100  fort.14
ln -s ~/lines/gf0500.100  fort.15
ln -s ~/lines/gf0600.100  fort.16
ln -s ~/lines/gf0800.100  fort.17
ln -s ~/lines/gf1200.100  fort.18
~/width/inpwidth.exe
mv fort.1 inpwidth.dat
rm fort.*


 
In inpwidth.com the input atomic line lists are mostly the Kurucz line lists, 
but they are not exactly the same. See Castelli & Hubrig, 2004, A&A 425, 263
